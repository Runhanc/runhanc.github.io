Introduction of URDF

Overview

Unified Robot Description Format

URDF将机器人描述为一棵由Joints连接的tree of Links。Link代表机器人的物理组件，而Joint则代表一个Link如何相对于另一个Link移动，有效地定义了Link在空间中的位置。

Advantages

  1. Perform collision checking

  2. Understand robot kinematics

  3. Perform transformation math

指定一个link/joint的两个主要原因：

  1. 机器人的一个部件相对于另一个部件运动（如机械臂的每一段）

  2. 相对于另一个部件不运动的部件，但方便有自己的参考点和变换（常见的例子是传感器的位置，如摄像头或激光雷达）

因此，通常会在URDF中制定joint的类型，常见的类型如下：

  1. Revolute: 旋转运动，有最大/最小的角度限制

  2. Continuous:无限制的旋转（比如轮子）

  3. Prismatic：线性滑动运动，具有最小/最大位置限制

  4. Fixed：child link与parent link刚性连接

Syntax

The robot tag and XML declaration

对于 URDF 文件来说，根标签就是机器人标签，这里唯一需要注意的是，我们可以设置 name 属性。

Link tags

  • Visual - 这里定义了在RViz和Gazebo会显示的东西

    • Geometry - box/cylinder/sphere, 并且有对应的尺寸参数，或者mesh

    • Origin - 几何偏移量，使其无需以链接原点为中心

    • Material - 基本上就是颜色。我们可以指定已声明材质的名称，也可以直接描述颜色。(请注意，这将在 RViz 中设置颜色，但不会在 Gazebo 中设置）

  • Collision - 用于物理碰撞的计算

    • Geometry and Origin - 选项与视觉效果相同。这通常是从视觉标记复制粘贴过来的，但出于计算原因，我们可能需要一个更简单的碰撞几何体（例如用方框代替网格）

  • Inertial - 这也用于物理计算，但决定了link对力的响应。惯性属性包括：

    • Mass - link的质量

    • Origin - 重心，这是link可以平衡的点，大部分简单情况下，这只是中心点

    • Inertia - 转动惯量矩阵。 它描述了质量分布将如何影响旋转。这里有一份常见形状的矩阵列表。 将我们的链接近似为棱柱、椭圆体或圆柱体，可以实现相当精确的模拟，但对于大多数用途来说，非常粗略的猜测即可。

Joint tags

虽然我们通常认为机器人是由链接组成的，但实际上，关节才是机器人结构的所有细节所在，因为它们定义了链接的位置，以及它们之间的相对移动方式。

  • Name - 关节的名称。如果没有这个，有些东西还能用，但有些东西就不能用了。为了以防万一，最好给它们都起个名字

  • Type - 如前所述的Joint类型（最常用的四种接头类型是fixed, prismatic, revolute, and continuous）。

  • Parent and child links - 这个Joint连接的Link的关系

  • Origin - 在施加任何运动之前，两个Links之间的关系

对于固定Joints，这些参数就足够了，但对于非固定Joints，我们通常需要指定一些其他特性。可供选择的参数有很多，但最常见的有以下两个:

  • Axis - 沿哪个轴或绕哪个轴移动

  • Limits - 系统其他部分可能预期的物理执行限制。这些限制可能包括

    • Upper and Lower position limits - in metres/radians

    • Velocity limits - in m/s or rad/s

    • Effort limits - in N or Nm
