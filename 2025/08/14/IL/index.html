Imitation Learning

行为克隆

原理

较为直观：

  • 如果人类能完成一个任务，我们就记录下观察到的状态（state）和对应的动作（action），然后训练一个模型去拟合“状态 → 动作”的映射。

  • 训练好的模型就可以在没有人类示范的情况下自主执行相似任务。

数学解释：

  1. 机器人任务可表示为一个策略（policy）πθ(a|s)，即在状态 s 下选择动作 a 的概率分布。

  2. 专家演示提供了数据集𝒟 = {(si, ai)}i = 1N。

  3. 训练过程是最小化监督学习损失：


    θ* = arg minθΣ(si, ai) ∈ 𝒟ℒ(πθ(si), ai)

  • 对离散动作空间可用交叉熵损失

  • 对连续动作空间可用均方误差 (MSE)

  • 训练完成后，直接用 πθ(s) 决策即可。

核心本质：

  • 把机器人决策学习转化为一个回归/分类问题

  • 不依赖环境奖励（与强化学习不同）

  • 性能依赖于演示数据的质量与覆盖范围，泛化能力较差，并且无法很好地从错误中恢复

DAgger算法

Dagger（Dataset Aggregation，数据集聚合）是一种用于**模仿学习（Imitation Learning）**的算法，常用于机器人和强化学习领域。它的核心思想是：结合专家演示和智能体自身的探索数据，不断迭代优化策略，从而克服传统行为克隆（Behavior Cloning）容易出现的“分布偏移”问题。

Dagger 算法流程简述

  1. 初始化：用专家演示数据训练一个初始策略（通常是行为克隆）。

  2. 交互采样：用当前策略与环境交互，收集状态。

  3. 专家标注：让专家为这些状态提供正确动作（标签）。

  4. 数据聚合：将新采集的（状态，专家动作）对加入训练集。

  5. 策略更新：用聚合后的数据集重新训练策略。

  6. 重复步骤2-5，直到收敛。

主要优点

  • 缓解分布偏移：智能体会遇到自己策略下的新状态，专家会为这些新状态提供标签，避免“训练-测试分布不一致”。

  • 数据高效：只需少量专家干预即可获得较好性能。

逆向强化学习

给定专家演示的最优策略下，反推出其背后的奖励函数

算法流程

  1. 收集专家数据：收集由专家在执行任务时产生的行为轨迹数据（状态-动作序列）

  2. 初始化奖励函数： 算法会从一个初始的奖励函数猜测开始。这个奖励函数通常用一组参数来表示。

  3. 迭代优化循环：

  • 步骤A (RL步骤): 基于当前的奖励函数，使用标准的强化学习算法（如Q-learning或Policy Gradient）来求解一个最优策略。

  • 步骤B (比较与更新): 比较上一步学到的策略与专家策略的差异。如果两者相似，说明当前的奖励函数可能就是正确的。如果差异很大，则调整奖励函数的参数，使得在该奖励函数下，专家策略的表现优于其他任何策略。

  • 循环与收敛： 重复步骤A和B，直到学到的策略与专家的策略非常接近，此时算法收敛，得到的奖励函数即为最终结果。

相关算法

Apprenticeship Learning / Max-Margin IRL（特征期望匹配）

  • 假设奖励函数是状态特征的线性组合：


    Rw(s) = w⊤ϕ(s)

这里：

  • ϕ(s) 是一个 特征向量（比如速度、距离目标的远近、能量消耗等）。

  • w是待学习的权重参数，表示奖励函数的偏好。

  • 我们不能直接从专家演示中看到奖励R，但是可以统计专家访问到的状态特征的“平均值”（加上折扣因子），即特征期望：


μ(π) = 𝔼π[Σtγtϕ(st)]

  • 这其实就是特征访问频率的加权和。

因此：

  • 找到一个w，让专家的策略$ _E$的回报高于所有其他策略：


w⊤μ(πE) ≥ w⊤μ(π),  ∀π

  • 直觉上，就是让w认为专家比任何策略都“好”，并且拉大这个差距（Max-Margin）。

流程：

  1. 估计专家的特征期望：

  • 迭代更新：

    • 给定当前w，用 RL 求解最优策略πw。

    • 得到它的特征期望μ(πw)。

    • 如果 μ(πw)很接近μE，就停。

    • 否则更新w，或者把μ(πw)加到候选策略集合中继续优化。

  • 停止条件：

    • ∥μ(πw) − μE∥ < ϵ。

特点：

  • 匹配特征分布 ≈ 让智能体在环境中访问的状态分布和专家相似。

  • 这样学到的奖励函数可以泛化到没有见过的状态。

Maximum Entropy IRL（最大熵 IRL）

  • 在满足“专家最优”的同时，避免对策略做不必要的偏好。

  • 传统 IRL 找到的奖励可能有很多个，最大熵原则可以让我们在这些奖励中选择最均匀、最随机的轨迹分布（信息量最大）。

假设轨迹（路径）ζ 的奖励是其状态特征的线性组合：


reward(fζ) = θTfζ = Σsj ∈ ζθTfsj

其中：

  • fsj是状态 sj的特征向量

  • θ是模型需要学习的奖励权重

  • fζ是整条路径的特征计数

目标是使学习者的行为与观察到的专家行为的特征期望相匹配：∑PathζiP(ζi)fζi = f̃

学习奖励权重（最大似然估计）:

为了学习最优的奖励权重θ，论文采用最大似然估计。目标是最大化观察到的专家演示数据在最大熵分布下的似然函数

该函数的梯度计算为经验特征计数与学习者预期特征计数之间的差值 ：


∇L(θ) = f̃ − ∑ζP(ζ ∣ θ, T)fζ = f̃ − ∑siDsifsi

其中Dsi是预期状态访问频率。在最大值处，特征期望匹配，从而保证学习者的行为与专家演示行为等效。

高效状态频率计算

计算梯度所需的预期状态访问频率Dsi是一个计算密集型任务。论文提出了一种高效算法，类似于条件随机场（CRFs）中的前向-后向算法，避免了对所有可能路径的显式枚举，从而使该方法能够应用于大型状态空间。

该算法通过反向传播（计算配分函数）和前向传播（计算状态访问频率）两个阶段来高效地计算Dsi。

特点：

  • 能自然处理次优专家（因为分布是随机的，不会强制唯一最优）。

  • 学到的奖励函数在不同策略间不会武断区分。

缺点

  1. 计算成本高： IRL的迭代过程中内嵌了一个完整的RL求解过程，这使得其计算复杂度通常远高于标准RL。

  2. 奖励函数的模糊性： 对于同一组专家数据，可能存在多个不同的奖励函数都能很好地解释其行为。

  3. 对专家最优性的假设： IRL通常假设专家的演示是（近乎）最优的。如果专家演示质量不高或包含错误，学习效果会大打折扣。

  4. 需要大量高质量数据： 和许多机器学习方法一样，IRL需要充足且有代表性的专家数据。

GAIL

GAIL（Generative Adversarial Imitation Learning，生成对抗模仿学习）把逆强化学习（IRL）问题转化成了一个生成对抗网络（GAN）的问题，从而直接学策略而不需要显式地恢复奖励函数。

核心思想

GAIL 借鉴了 GAN 的结构：

  • 生成器 G → 在 GAIL 中就是策略 πθ(a|s) 输入状态s，输出动作分布，生成轨迹。

  • 判别器 D → 用于判断一对(s, a)是来自专家还是策略。 Dω(s, a) ∈ (0, 1)，越接近 1 表示越像专家。

  • 策略的目标：愚弄判别器，让判别器无法区分轨迹来自专家还是自己。

数学公式

GAIL 的优化目标类似 GAN 的 min-max 博弈：


minπθmaxDω  𝔼(s, a) ∼ πE[log Dω(s, a)] + 𝔼(s, a) ∼ πθ[log (1 − Dω(s, a))] − λH(πθ)

解释：

  • 第一项：让判别器对专家样本输出高分（接近 1）

  • 第二项：让判别器对生成的轨迹输出低分（接近 0）

  • 第三项：策略熵正则化，鼓励探索

一般写作：H(πθ) = Eτ ∼ πθ[ − Σtlogπθ(at ∣ st)]

  • λ > 0：控制熵项权重

奖励信号的替代 在训练策略时，GAIL 把判别器输出转换为奖励：

r̂(s, a) =  − log (1 − Dω(s, a))

这样 RL 就能用这个奖励优化策略。
